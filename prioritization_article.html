Automation Priority Builder

In this nerdy spreadsheets-heavy example I'll show how we prioritize test cases for the automation coverage

1. We start with simple Assumptions:

Each Test Case should have properties:
● 'System Area' as a main grouping characteristics.
● 'Risk' for the application if test case fails.
● 'Manual Effort' required when testing it.
● 'Estimate' to turn it into auto-test.

The simple idea behind this:
Tests having highest 'Risk' and 'Manual Efforts' but which are easier to cover are top prioritized, since we consider them as a valuable low-hanging fruits.

2. Export data

Test Cases exported into Google Spreadsheets.

3. Create a ruleset

Combination of System Risk value and Manual Effort should then be considered by the team.
Subjectively team assigns the Automation Priority Result to each combination.
This way we get a ruleset which can later be used to mark test cases.

4. Applying ruleset

Agreed ruleset is used to mark each test case.
This is done with some wizardy around Google Spreadsheets formulas. Mostly all work is done with INDEX() and MATCH() functions.

5. Grouping cases by the area

In the next step, we'll group the test cases by the System Area property.
This is done on a separate sheet.
We will use UNIQUE() function to get a list of all possible System Areas.
We will also count total number of cases in each group.

6. Calculating other group parameters

Now that we have groups in place, we can also calculate some group-specific attributes.
Like number of test cases already covered, or the number of cases considered Highest/High/Medium/Low. We will also summarize the time needed to cover the whole area.
This work is mostly done with COUNTIF() and QUERY() functions.

7. Applying Weights

Now this part is bit math-nerdy.
When calculating the total 'Rating' for the System Area, we would like to have Highest and High test cases to have more weight, than Medium and Low.
That's why we apply Weight Multipliers:
● Highest = 0,7
● High = 0,3
● Medium = 0,1
● Low = 0,05

Now the only thing left is to divide the weighed System Area rating by the time needed to cover it with test cases.
And we get our final rating:

8. Demoing results

The final step of this drama is to show rating to the business team.
We've prepared another sheet where the System Areas are filtered ascending, based on the calculated rating.
Since this process was done iteratively, we've also calculated the coverage percent for each System Area. We've already calculated total number of cases and a number of covered test cases for each System Area, so was a simple task.
This was our regular report of progression.
