Tables Automation

This article will be dedicated to the semantic problem around tables in Web UI

1. The Problem:

Creating Automation for tables in Web UI is quite challenging from semantical standpoing.
Usually it's quite hard to write readable tests when the steps or assertions require some actions in the table.

2. Preparation and Assumptions

For this article I've prepared a simple framework built upon Playwright, nUnit and Specflow in C#. This framework is public free and available in Samples-TablesAutomation repository.
This framework implies parallel test runs (by features). It has appsettings configuration file, allowing users to select from headful/headless browser screenshots, capture screenshots, write traces and vides. Overall, this framework can be considered a draft for test automation setup.
In the article I'll be focusing only on the particular challenges and questions.

3. Target

I'll be using material.angular.io table overview page for these examples.

This page has good examples:
● Simple table with headers
● Table with sorting
● Table with clickable rows

4. Why SpecFlow?

It can be very tricky to create semantics around tables. Here are some challenges:
● Check if table has containing cells with particular texts.
● Click into a row, containing cells with particular texts.
● Check if table has proper headers.
● Sort table and get all values from particular column.

SpecFlow has an inbuilt support for Vertical and Horizontal tables (see more here).
This provides us with a clear semantics.
